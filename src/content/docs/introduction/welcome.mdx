Welcome to the documentation portal for **OSIRIS JSON**, The Open Specification for Infrastructure Resource Interchange Schema designed to allow you to take snapshots of your infrastructure at a point in time. 

Modern enterprise environments suffer from extreme infrastructure data fragmentation. When managing heterogeneous systems across public clouds, private virtualization, on-premise hardware, and operational technology (OT), organizations often face high tool integration costs, documentation drift, and the risks of tribal knowledge silos. OSIRIS JSON addresses these challenges by offering a lightweight, vendor-neutral schema to normalize and capture the state of your infrastructure at a point in time.

## Documentation Structure

Our documentation is divided into five main sections:

<Steps>
    1. **Introduction & Core Concepts**: High-level overviews, architectural value propositions, and real-world use cases.
  2. **Producers & Integration**: Guides on how to execute OSIRIS JSON Producers locally or in CI/CD to capture snapshots from AWS, Azure, Cisco, and more.
  3. **Consumers & Tooling**: Tools that ingest OSIRIS JSON documents to generate visual topologies, reports, and change drift logs.
  4. **Validation**: An explanation of the three-level validation pipeline to guarantee metadata and relational integrity.
  5. **Developer & Contributor Space**: Specifications, guidelines, and SDK details for building on top of the OSIRIS format.
</Steps>
